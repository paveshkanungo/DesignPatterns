# 📌 What are Design Patterns?

**Design Patterns** are reusable solutions to common software design problems.

Instead of solving the same design problems repeatedly, developers can use **proven templates** that improve:

- Code maintainability
- Reusability
- Scalability
- Flexibility

The most widely known catalog of design patterns was introduced in the book:

> **Design Patterns: Elements of Reusable Object-Oriented Software**  
> by *Erich Gamma, Richard Helm, Ralph Johnson, and John Vlissides*  
> (also known as the **Gang of Four**)

---

# 🧩 Design Pattern Categories

Design patterns are divided into **three main categories**.

---

# 🏗 Creational Patterns

Creational patterns deal with **object creation mechanisms** and help make systems independent of how objects are created.

| Pattern          | Description                                                 |
|------------------|-------------------------------------------------------------|
| Singleton        | Ensures only one instance of a class exists                 |
| Factory Method   | Creates objects without exposing instantiation logic        |
| Abstract Factory | Produces families of related objects                        |
| Builder          | Builds complex objects step by step                         |
| Prototype        | Creates objects by cloning existing instances               |

---

# 🧱 Structural Patterns

Structural patterns deal with **class and object composition**.  
They help ensure different parts of a system work together efficiently.

| Pattern   | Description                                            |
|-------    |--------------------------------------------------------|
| Adapter   | Converts one interface to another expected interface   |
| Bridge    | Separates abstraction from implementation              |
| Composite | Treats objects and compositions uniformly              |
| Decorator | Adds behavior dynamically to objects                   |
| Facade    | Provides a simplified interface to a complex subsystem |
| Flyweight | Reduces memory usage by sharing objects                |
| Proxy     | Controls access to another object                      |

---

# 🔁 Behavioral Patterns

Behavioral patterns focus on **communication between objects** and how responsibilities are distributed.

| Pattern                 | Description                                                |
|-------------------------|------------------------------------------------------------|
| Strategy                | Encapsulates interchangeable algorithms                    |
| Observer                | Defines dependency between objects for notifications       |
| Command                 | Encapsulates a request as an object                        |
| Chain of Responsibility | Passes request along a chain of handlers                   |
| State                   | Changes object behavior when state changes                 |
| Template Method         | Defines skeleton of an algorithm                           |
| Iterator                | Provides sequential access to elements                     |
| Mediator                | Centralizes communication between objects                  |
| Memento                 | Captures and restores object state                         |
| Visitor                 | Separates algorithms from object structures                |
